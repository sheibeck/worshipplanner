Improve the SongSlotPicker dropdown so users can browse more rotation songs without leaving the screen, and surface each song's team tags inline on the picker rows.

<current_constraints>
Two factors currently limit how many rotation songs the user sees:

1. **Panel height cap:** the dropdown has `max-h-80` (Tailwind = 20rem ≈ 320px) on the panel `<div>` (line 36).  Because the panel also contains the sticky search bar (~50px), the AI Picks section (header + up to 3 rows ≈ 130px), and a divider, the rotation list typically only has room for ~5 visible rows before scrolling.

2. **Hard slice in `suggestions` computed (line 211–212):**
   ```ts
   const suggestions = computed<SuggestionResult[]>(() => {
     const results = rankSongsForSlot(props.songs, props.requiredVwType, props.serviceTeams)
     return results.slice(0, 5)   // ← caps the rotation list at 5 regardless of panel height
   })
   ```
   Even if the panel were taller, the user would still only see 5 rotation songs.

Both must change to deliver the requested improvement. The dropdown's adaptive positioning logic in `openDropdown()` (lines 258-303) uses the same 320px cap (`const maxH = 320`); both numbers must move together so positioning math stays consistent.
</current_constraints>
