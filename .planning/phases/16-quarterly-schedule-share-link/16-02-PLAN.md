<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| unauthenticated public → orgSlugs / quarterShares | Anyone can read these collections (public share); only signed-in users may write |
| org name (user-controlled) → Firestore doc ID | Slug becomes part of a document path — must be sanitized before use |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-02-01 | Tampering | doc-ID injection via unsanitized org name (`/`, `..`) | mitigate | deriveSlug strips to `[a-z0-9-]+` client-side BEFORE any doc() path construction (Task 1); rules test asserts writes only from signed-in users |
| T-16-02-02 | Spoofing | slug-squatting a reserved word to hijack app navigation | mitigate | RESERVED_SLUGS pre-filter in claimSlug rejects/suffixes the 10 reserved segments before any write (D-19) |
| T-16-02-03 | Elevation of Privilege | unauthenticated write to public collections | mitigate | Rules gate create/update to `isSignedIn()`; orgSlugs update/delete `if false` (create-once) |
| T-16-02-04 | Information Disclosure | slug/quarter enumeration (guessing other churches' URLs) | accept | Explicitly accepted privacy tradeoff — memorable URL is intentionally guessable (SPEC constraint, locked round 2) |
| T-16-02-05 | Information Disclosure | quarterShares leaking PII beyond names | mitigate | quarterShares snapshot carries only pre-resolved names (D-24) — no email/phone; enforced by the finalizeAndShare writer (plan 16-09), rules only gate write auth |
| T-16-02-06 | Tampering | slug-collision race (two orgs claim same slug) | mitigate | Firestore create-only rule (`resource == null`) makes first-writer-wins server-side; claimSlug retries with numeric suffix on permission-denied — no Cloud Function/transaction needed |

No package installs in this plan — supply-chain (T-16-SC) not applicable.
