    - `notes` = ''
    - `arrangements` = input arrangements mapped to `{ id: arr.id, name: arr.name, key: '', bpm: null, lengthSeconds: null, chordChartUrl: '', notes: '', teamTags: [] }`
    - `hidden` = false

    - First call: `GET /api/planningcenter/services/v2/songs?include=tags&per_page=100`
    - Auth: `Authorization: 'Basic ' + btoa(appId + ':' + secret)`
    - Follow `response.links.next` if present (absolute URL — use directly, do NOT prepend PC_BASE_URL)
    - Collect all `data` items and all `included` (tags) across pages

    ```typescript
    export async function fetchAndMapPcSongs(
      appId: string,
      secret: string,
    ): Promise<UpsertSongInput[]>
    ```
    Logic:
    1. Call `fetchAllPcSongs(appId, secret)` to get `Array<{ song, tags }>`
    2. For each song (in batches of 10 to avoid rate limits): call `fetchSongArrangements(appId, secret, song.id)` (imported from `@/utils/planningCenterApi`)

    4. Return the resulting `UpsertSongInput[]` — no store calls, no Firestore writes
    This is the function PcImportModal calls to get data for preview before the user confirms import.

    3. Count added vs updated by comparing against `store.songs` before upsert
    4. Return `{ added, updated, errors }`
