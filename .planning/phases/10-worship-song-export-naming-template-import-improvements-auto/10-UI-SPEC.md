## Component Inventory

### 1. PC Teams Checkbox Group (new — inside export dialog)

Location: Export dialog in `ServiceEditorView.vue`, between the Template selector and the Service Date read-only field.

Structure:
```
<div class="mb-3">
  <label class="block text-xs text-gray-400 mb-2">Teams</label>
  <!-- loading shimmer -->
  <div v-if="teamsLoading" class="text-sm text-gray-400">Loading teams...</div>
  <!-- team checkboxes -->
  <div v-else class="flex flex-wrap gap-3">
    <label v-for="team in pcTeams" class="flex items-center gap-2 cursor-pointer">
      <input
        type="checkbox"
        v-model="selectedPcTeamIds"
        :value="team.id"
        class="h-4 w-4 rounded border-gray-600 bg-gray-800 text-indigo-600
               focus:ring-indigo-500 focus:ring-offset-gray-900"
      />
      <span class="text-sm text-gray-200">{{ team.name }}</span>
    </label>
  </div>
  <!-- empty state -->
  <p v-if="!teamsLoading && pcTeams.length === 0" class="text-sm text-gray-500">
    No teams found for this service type.
  </p>
</div>
```

Checkbox style matches existing Teams section in `ServiceEditorView.vue` line 354-360 exactly: `h-4 w-4 rounded border-gray-600 bg-gray-800 text-indigo-600 focus:ring-indigo-500 focus:ring-offset-gray-900`.

Auto-pre-selection behavior: On fetch, auto-check any PC team whose name case-insensitively matches a WorshipPlanner service team name (from CONTEXT.md decision 3).

### 2. Orchestra-Dimmed Song Row (modified — SongSlotPicker.vue)

When `serviceTeams.includes('Orchestra')`:

- **Orchestra-tagged songs** (`song.teamTags.includes('Orchestra')`): render at full opacity with `text-gray-100` title — no class change from current.
- **Non-orchestra songs**: add `opacity-50` to the wrapping `<button>` element for both "By Rotation" and "Search Results" sections.

Pattern (from CONTEXT.md discretion area): Use `opacity-50` on the button wrapper. This dims title, key, last-used meta, and badge together — no partial dimming.

Example class binding:
```
:class="isNonOrchestra(result.song) ? 'opacity-50' : ''"
```

No tooltip, no text label, no icon — opacity only. Consistent with `ghostClass: 'opacity-30'` drag-ghost pattern already in the view.

### 3. Updated Existing Plan Info Copy (modified — export dialog)

Replace line 288 copy from:
> "Songs replace 'Worship Song' items and scriptures replace 'Scripture Reading' items in the plan. Any extras are appended at the end."

With:
> "Worship Song items are replaced. Scripture Reading items are replaced. Unmatched placeholders are removed."

Style unchanged: `text-xs text-gray-500 mb-3`.

## Interaction States

### Export Dialog — Teams Section

| State | Visual |
|-------|--------|
| Loading (fetching teams) | `text-sm text-gray-400` text "Loading teams..." inline — no spinner (stays consistent with `exportLoading` pattern) |
| Teams loaded, none match service | All unchecked, checkboxes visible |
| Teams loaded, auto-matched | Matched teams pre-checked on render |
| Team-add failure (non-fatal) | Inline `text-xs text-amber-400` note below export result, not blocking |
| No teams returned | Empty state `text-sm text-gray-500` inline |

### Song Picker — Orchestra Mode

| State | Visual |
|-------|--------|
| Orchestra in serviceTeams, song is orchestra-tagged | Normal rendering — `text-gray-100` title, full opacity |
| Orchestra in serviceTeams, song is NOT orchestra-tagged | `opacity-50` on button wrapper |
| Orchestra NOT in serviceTeams | No change — all songs render normally |
| Search mode (Orchestra active) | Same opacity rules apply to search results |
