export interface ResolvedCsvPerson {

  standing: { name?: string; email?: string; phone?: string; roles?: string[]; frequencyTargetN?: number }

export const useQuartersStore = defineStore('quarters', () => {

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Firestore (org data) | Quarter + personQuarterData PII (blackout dates, pairings) written under organizations/{orgId} |
| Firestore shareTokens → public | Finalized quarterSnapshot readable by anyone with the token (unauthenticated) |
| Cross-org | Quarter reads/writes must stay within the signed-in editor's org |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-06-01 | Information Disclosure | Public share token guessing/enumeration | mitigate | 144-bit (18-byte) crypto.getRandomValues hex token — same entropy already accepted for service shares; existing `shareTokens` rule restricts create to signed-in editors and update/delete to false (RESEARCH Security) |
| T-13-06-02 | Information Disclosure | Public snapshot leaking more PII than needed (emails/phones) | mitigate | quarterSnapshot denormalizes ONLY names + roles + calendar — no emails, phones, or blackout dates written into the public shareTokens doc |
| T-13-06-03 | Tampering | Quarter reset wiping standing person fields | mitigate | applyCsvToQuarter writes blackouts/pairings only to quarter.personQuarterData; standing fields are upserted via the roster store, never overwritten with quarter-scoped values (Pitfall 3, test-enforced) |
| T-13-06-04 | Information Disclosure | Cross-org quarter access | mitigate | All quarter reads/writes scoped to organizations/{orgId}; existing org-editor Firestore rule covers the new `quarters` collection (no rules change) |
| T-13-06-05 | Repudiation/Tampering | Unexpected loss of manual edits on regenerate | mitigate | Two explicit modes — 'fillGaps' preserves existing assignments; 'regenerate' overwrites and the UI (Plan 08) gates it behind a destructive confirmation (UI-SPEC copywriting) |
</threat_model>
