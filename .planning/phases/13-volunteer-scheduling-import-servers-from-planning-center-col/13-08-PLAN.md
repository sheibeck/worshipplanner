<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CSV file → app UI → Firestore | Untrusted leader-supplied CSV parsed and applied to quarter PII (blackouts, pairings) |
| Name reconciliation | CSV names mapped to roster person ids — wrong mapping corrupts a person's quarter data |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-08-01 | Tampering | Silent wrong-person CSV match corrupting quarter data | mitigate | matchNameToPerson surfaces every non-exact row; commit requires explicit map-to-existing/create-new per unmatched/ambiguous row (D-16) — no silent auto-create |
| T-13-08-02 | Input Validation | Malformed CSV cells / out-of-quarter blackout dates | mitigate | parseVolunteerCsvRow warnings + expandBlackoutCell drops out-of-quarter dates with a visible row warning; the modal renders warnings before commit rather than corrupting data |
| T-13-08-03 | Information Disclosure | Quarter setup visible to non-editors | mitigate | /schedule route `requiresEditor: true`; store writes gated by org-editor Firestore rule |
| T-13-08-04 | Repudiation | Accidental destructive regenerate wiping manual edits | mitigate | Regenerate gated behind the explicit destructive confirmation; Fill Remaining Gaps offered as the non-destructive default (UI-SPEC) |
| T-13-08-05 | Tampering (CSV formula injection) | Excel-formula cells in import | accept | PapaParse parses cells as data; no CSV re-export exists this phase (RESEARCH Security) |
</threat_model>
