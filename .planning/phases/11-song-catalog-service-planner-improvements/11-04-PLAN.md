<objective>
Make the service-planner song picker type-agnostic and fully browsable: remove the 1/2/3 type filter and type-match ranking so any song can be picked for any slot (D-09, D-10), broaden AI Picks (D-11), fix the "stops after a handful" browsability with IntersectionObserver batching on both the rotation and search lists (D-12), show themes/user-tag pills (D-06 picker side), and add include/exclude tag filtering in the picker (D-03 picker side). The VW badge stays as info only.

Purpose: The user applies the Vertical Worship 1-2-3 paradigm themselves; the picker should present the whole catalog without app-enforced type constraints, and scrolling must reach every song.
Output: A SongSlotPicker that browses the full non-hidden catalog, ranks purely by rotation/search relevance, filters by tag, and shows all metadata pills.

This plan is NOT autonomous — Task 3 ends in a human-verify checkpoint for the picker browsing/filtering UX.
</objective>
