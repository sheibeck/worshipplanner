| User tag input → Firestore/store | User-entered tag strings persisted and later rendered |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Tampering | upsertSongs import merge (songs.ts) | mitigate | Preservation override sets `tags: existing.tags ?? []` explicitly so a crafted/empty PC import payload cannot clear user tags; themes union cannot delete existing themes |
| T-11-02 | Information Disclosure | TeamTagPill render | accept | Tag/theme strings rendered via Vue interpolation `{{ tag }}` (auto-escaped); no v-html introduced. No PII in tag fields |
| T-11-03 | Elevation of Privilege | pcSongImport tag write path | accept | tags always set to `[]` from import — import can never source a tag value; only authenticated in-app edits (existing Firestore ownership rules) write tags. No new write path added |
</threat_model>
