Structure tests in describe blocks: "validatePcCredentials", "fetchServiceTypes", "createPlan", "createItem", "addSlotAsItem", "buildPlanTitle".

    **Implementation file:** Create `src/utils/planningCenterApi.ts` with these exports:

    1. `PC_BASE_URL` — use `import.meta.env.DEV ? '/api/planningcenter/services/v2' : 'https://api.planningcenteronline.com/services/v2'`

    2. `basicAuthHeader(appId, secret)` — internal helper, returns `'Basic ' + btoa(appId + ':' + secret)`

    3. `validatePcCredentials(appId, secret)` — GET `/service_types?per_page=1`, returns `{valid: boolean, error?: string}`. 200=valid, 401=invalid credentials, other=API error, catch=network error.

    4. `fetchServiceTypes(appId, secret)` — GET `/service_types?per_page=100`, returns `Array<{id: string, name: string}>`. Parse JSON:API response `json.data` array mapping `{id: st.id, name: st.attributes.name}`.

    5. `createPlan(appId, secret, serviceTypeId, title, dates?)` — POST to `/service_types/{serviceTypeId}/plans` with JSON:API body `{data:{type:'Plan',attributes:{title, ...(dates ? {dates} : {})}}}`. Returns `json.data.id`. Throws on non-ok with status + response text.

    6. `createItem(appId, secret, serviceTypeId, planId, params: {title, itemType: 'song_arrangement'|'regular', description?, sequence?})` — POST to `/service_types/{serviceTypeId}/plans/{planId}/items` with JSON:API body. Use `html_details` for description field. Returns `json.data.id`. Throws on non-ok.

    7. `buildPlanTitle(service: Service)` — Pure function. If sermonPassage exists, use `formatScriptureRef(service.sermonPassage)` (import from planningCenterExport.ts). Else if service.name non-empty, use that. Else "Service". If teams.length > 0, append ` (teams.join(', '))`.

    8. `addSlotAsItem(appId, secret, serviceTypeId, planId, slot, sequence, songs)` — Maps each SlotKind to a createItem call per locked decisions:
       - SONG: title = "songTitle (Key: songKey)" or "[Empty Song]", itemType = 'song_arrangement'
       - HYMN: title = "hymnName #hymnNumber" (omit # if no number), itemType = 'song_arrangement'
       - SCRIPTURE: title = "Book Chapter:VerseStart-VerseEnd", description = await fetchPassageText(ref) (catch errors silently), itemType = 'regular'
       - PRAYER: title = "Prayer", itemType = 'regular'
       - MESSAGE: title = "Message", itemType = 'regular'

    All fetch calls use headers: `{Authorization: basicAuthHeader(appId, secret), Accept: 'application/json'}`. POST calls add `'Content-Type': 'application/json'`.

    Per user decision: songs array parameter is needed for SONG slots to resolve songId to title/key from the slot snapshot data (which is already denormalized on the slot itself — songTitle and songKey fields). So for SONG slots, read directly from slot.songTitle and slot.songKey. The `songs` parameter is available but not strictly needed since data is denormalized; keep the parameter for future flexibility but primarily use slot snapshot data.
