<objective>
Replace the single "Serve frequency" select in the Edit Volunteer form with one cadence control per role the person holds (D-01), defaulting blanks to monthly N=4 (D-02).

Purpose: This is the user-facing point of the per-role frequency model — a leader sets Guitar weekly and Vocals monthly for the same person. The form iterates the person's checked roles and renders a cadence `<select>` per role, backed by a reactive `roleFrequencies` object, reusing the exact `nToFrequencyLabel` option markup already present. On load, roles missing a per-role value default to the person's retained `frequencyTargetN` (D-03 migration expressed at read time), then to 4. Save persists the map via the store (persistence landed in 15-03).

Output: per-role cadence UI + reconciled frequency sort key in RosterView.vue, plus a new RosterView.test.ts (no existing test file for this view).
</objective>
