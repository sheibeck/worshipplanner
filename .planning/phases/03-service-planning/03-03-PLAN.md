  <name>Task 2: SongSlotPicker dropdown with suggestions and search (Teleport to body)</name>
  <files>src/components/SongSlotPicker.vue</files>
  <action>
    Create `src/components/SongSlotPicker.vue` — the in-slot dropdown that appears when a user clicks an empty song slot.

    **Props:**
    - `requiredVwType: VWType` — the VW type this slot requires
    - `serviceTeams: string[]` — active teams for filtering
    - `currentSongId: string | null` — currently assigned song (to show "Change" vs "Select")
    - `songs: Song[]` — full song array from songStore (passed down, not imported directly)

    **Emits:**
    - `select(song: { id: string, title: string, key: string })` — when user picks a song
    - `clear()` — when user removes the assigned song

    **Behavior (per CONTEXT.md locked decisions):**
    1. When user clicks the slot area, toggle dropdown open/closed
    2. On open, call `rankSongsForSlot(songs, requiredVwType, serviceTeams)` to get suggestions
    3. Show top 5 suggestions in the dropdown
    4. Each suggestion row displays:
       - Song title (text-gray-100)
       - Preferred key from first arrangement (text-gray-400, small)
       - "Last used X weeks ago" or "Never used" (text-gray-500, small)
       - SongBadge component for VW type (reuse existing)
       - If `isRecent` is true, show a subtle "Recent" label (amber text) — deprioritized but visible
    5. Search bar at top of dropdown: type to search the full library filtered to correct VW type
       - When search is active, show matching songs below the suggestions section
       - Search filters by title (case-insensitive) from the already-filtered-by-VW-type list
    6. Click a suggestion/search result to emit `select` with `{ id, title, key }` (key from first arrangement)
    7. If a song is already assigned, show the song info with a "Change" button and an "x" clear button

    **CRITICAL — Dropdown positioning (Teleport to body):**
    Per RESEARCH.md Pitfall 4: AppShell uses `overflow-y-auto` on the main content area, which creates a stacking context that clips `position: absolute` dropdowns. Use `<Teleport to="body">` for the dropdown panel — same pattern as SongSlideOver.vue.

    Implementation:
    - The slot area itself is the trigger element (click to open)
    - When open, compute the trigger element's bounding rect via `getBoundingClientRect()`
    - Teleport the dropdown to `<body>` using `<Teleport to="body">`
    - Position the dropdown with `position: fixed; top: {rect.bottom}px; left: {rect.left}px; width: {rect.width}px`
    - Add a transparent backdrop overlay (`fixed inset-0 z-30`) that closes on click (same pattern as SongSlideOver.vue)
    - Dropdown panel itself gets `z-40` (above backdrop)
    - Tailwind classes on dropdown panel: `fixed z-40 bg-gray-800 border border-gray-700 rounded-lg shadow-xl max-h-80 overflow-y-auto`
    - Recalculate position on scroll/resize if needed (or accept fixed position is close enough for MVP)

    **Click-outside to close:** The transparent backdrop overlay handles this — clicking the backdrop emits close.

    **Dark mode:** gray-800 dropdown background, gray-700 borders, gray-900 hover on rows, text-gray-100 song titles, text-gray-400 secondary info.

    **Empty state:** If no suggestions match (e.g., no songs with the required VW type), show "No songs with this category. Add songs to your library first." with a link to /songs.
