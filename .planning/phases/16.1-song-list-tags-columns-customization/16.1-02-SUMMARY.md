# Phase 16.1 Plan 02: Church-level Vertical Worship mode flag Summary

**`vwModeEnabled` church-level flag (default ON) added to the auth store and Firestore org doc, with an editor-gated Settings toggle that mirror-writes the store on save.**

## Accomplishments
- `authStore.vwModeEnabled` is now the single source of truth every VW surface will gate on (D-16), defaulting to `true` and resetting to `true` on no-org, sign-out, and logout.
- `loadOrgContext` reads `organizations/{orgId}.vwModeEnabled` via the existing one-time `getDoc` pattern (no `onSnapshot`), with a `?? true` fallback for legacy org docs missing the field (D-15).
- Settings now has a "Vertical Worship" section with an editor-gated checkbox that calls `updateDoc` on the org doc and immediately mirror-writes `authStore.vwModeEnabled`, matching the `onSaveSlug` mirror-write template.
- Added targeted test coverage for `loadOrgContext`'s vwModeEnabled handling (default-true-on-missing-field, explicit-false, logout reset) — closing a pre-existing gap where `loadOrgContext` had no test coverage at all.

- Reverted the local checkbox value on a failed write so the UI never silently displays an unsaved state as saved.
