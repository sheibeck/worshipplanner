# Phase 16.1 Plan 06: Songs-panel tag fold + VW gating Summary

**Repointed SongsView's tag-filter union, SongFilters' VW-type dropdown, and the Add/Edit Song drawer to the folded single-tag model, hiding all VW indicators when church VW mode is off.**

## Performance

- **Duration:** ~15 min
- **Completed:** 2026-07-13T04:20:20Z
- **Tasks:** 2 completed
- **Files modified:** 3

## Accomplishments
- SongsView's `availableUserTags` computed now unions `themes ∪ tags` only — the redundant `teamTags` scan is gone, so the Songs-panel tag-filter checklist reflects the folded single tag set (D-12)
- SongFilters' VW-type `<select>` (All types / Type 1-3 / Uncategorized) is now wrapped in `v-if="authStore.vwModeEnabled"`, hiding it app-wide when the church has VW mode disabled (D-16)
- SongSlideOver's Add/Edit Song drawer no longer renders a dedicated "Team Tags" toggle section; the predefined names (Choir/Orchestra/Hymn) now seed the existing User Tags free-text type-ahead instead, so users can still quickly apply them as ordinary tags (D-01)
- SongSlideOver's VW Category selector is gated behind `authStore.vwModeEnabled`, matching the VW-off behavior established elsewhere in the phase (D-16 / research assumption A1)

## Files Created/Modified
- `src/views/SongsView.vue` - `availableUserTags` computed drops the `song.teamTags` scan line; now unions only `themes` and `tags`
- `src/components/SongFilters.vue` - Imports `useAuthStore`; VW-type `<select>` wrapped in `v-if="authStore.vwModeEnabled"`
- `src/components/SongSlideOver.vue` - Removed the "Song-level Team Tags" section, `form.teamTags`, `toggleSongTag`, and the teamTags-scanning `availableTags`/`songLevelTags` computeds; added a `tagSuggestions` computed (PREDEFINED_TAGS ∪ `songStore.allUserTags`) feeding the User Tags datalist; `onSave` no longer derives a teamTags union — it passes `props.song?.teamTags ?? []` through unchanged; VW Category selector wrapped in `v-if="authStore.vwModeEnabled"`

## Decisions Made
- `SongSlideOver.onSave` keeps writing a `teamTags` field (required by `Song`/`SongInput` types until plan 08 removes it) but stops deriving it from form/arrangement state — it now passes the prior `props.song?.teamTags` value through unchanged (empty array for new songs), consistent with how `hidden`, `pcSongId`, and `removedThemes` are already passed through for fields not editable in this form.
- Repurposed `PREDEFINED_TAGS` as datalist seed values (via the new `tagSuggestions` computed) rather than deleting the constant outright, satisfying the plan's instruction to keep Choir/Orchestra/Hymn quickly accessible as ordinary tags.
