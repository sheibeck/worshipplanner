## Team-tag fold scope (R-01)

First framing offered (fold depth) was rejected; the user reframed the intent: keep Orchestra/Choir as **service teams** that are ON for a given Sunday, but stop linking songs intrinsically to a team — songs should have exactly one flat tag set.

Reframed question: **Once songs are decoupled from teams, should scheduled teams still influence song suggestions?**

| Option | Description | Selected |
|--------|-------------|----------|
| Fully decoupled | Scheduled teams never touch the song list; remove Choir/Special-Service hard filter AND Orchestra +200 bonus | |
| Keep a soft nudge | No hard filtering; a scheduled team name matching a song tag ranks it slightly higher (soft bonus) | ✓ |
| Decide during planning | Lock the data-model merge now; defer scoring nuance | |

**User's choice:** Keep a soft nudge — soft only.
**Notes:** "The only team that dictates what songs we choose from are the days when the Orchestra team is playing. Since songs will have Orchestra tag on them, we can just manually figure out those... It's possible we'll have big choir numbers we can tag as Choir because we know they are good. Soft only." → merge `teamTags` into `tags`, remove the field, keep service team scheduling, remove hard filter, generalize the Orchestra bonus into a data-driven team-name↔tag soft nudge.
