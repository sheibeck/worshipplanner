# Phase 16.1 Plan 03: Decouple songs from service teams in ranking/search Summary

**rankSongsForSlot dropped its hard AND-logic team filter for a data-driven additive tag bonus, songSearch aliased `team:` to a plain tag match, and songMatchesQuery gained a vwModeEnabled flag that hides the `type:` prefix when VW mode is off.**

## Accomplishments
- `rankSongsForSlot` now returns every song unconditionally — service team scheduling is a nudge, never a gate (D-03)
- Replaced the hardcoded Orchestra-only bonus with a generalized, case-insensitive, additive per-team bonus over `song.tags`, supporting any team name and stacking across multiple scheduled teams (D-04)
- `team:` search prefix is now a documented alias for `tag:`, scanning `song.tags` instead of the now-legacy `song.teamTags` (D-06)
- `songMatchesQuery` accepts a `vwModeEnabled` flag (default `true`); when `false`, the `type:` prefix matches nothing, while all other prefixes and bare terms are unaffected (D-16)
