# Phase 16.1 Plan 01: Fold Team Tags Into Tags (Data Layer) Summary

**Read-fold of legacy `teamTags` into the flat `tags` set in the songs store, grow-only tag union + theme-removal-aware merge in `upsertSongs`, and PC import repointed to write team-style tags into `tags` instead of `teamTags`.**

## Accomplishments
- `Song.removedThemes: string[]` added, tracking user-removed themes for D-14
- `subscribe()` now folds legacy `teamTags` into `tags` at read time (deduplicated), so every consumer reading `song.tags` sees the merged set before any UI is repointed
- `upsertSongs` grows tags via de-duplicated union (existing ∪ incoming) instead of full-preserve — PC re-import adds team tags without dropping user tags (D-05)
- `upsertSongs` themes union existing+incoming then subtract `existing.removedThemes`, so a user-removed theme does not resurrect on re-import (D-14); `removedThemes` itself is preserved verbatim across re-import
- `mapPcSongToUpsert` now writes team-style tag names (non-category PC tags + "Orchestra" soft-nudge) into `tags`; `teamTags` is set to `[]` (field stays on the type until plan 08)
