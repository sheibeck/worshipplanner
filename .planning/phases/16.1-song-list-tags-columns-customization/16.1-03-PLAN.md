<objective>
Decouple songs from service teams in the ranking and search logic: remove the hard team filter, generalize the Orchestra +200 into a data-driven soft nudge over the folded `tags`, fold the `team:` search prefix into a plain tag match, and thread a `vwModeEnabled` flag into `songMatchesQuery` so the `type:` prefix hides when VW mode is off (D-16).

Purpose: D-03/D-04 change team scheduling from a gate to a nudge — all songs are always eligible, and a song tagged with a scheduled team's name simply ranks higher. D-06 retires the now-redundant `team:` prefix by aliasing it to `tag:`. D-16 requires the `type:` search prefix to be hidden app-wide when VW mode is off — since `songMatchesQuery` is a stateless pure function, the flag is threaded in as a parameter (default `true`) and the `case 'type':` branch returns no-match when it is false; the store and picker callers (both wired in plan 07) supply `authStore.vwModeEnabled`. All three are pure functions with existing test files, so this is a TDD plan.

Output: nudge-only `rankSongsForSlot`, aliased `team:` prefix, VW-gated `type:` prefix, and rewritten test suites.
</objective>
