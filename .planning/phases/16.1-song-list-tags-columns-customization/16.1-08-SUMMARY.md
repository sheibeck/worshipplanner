## Decisions Made

- Arrangement.teamTags stays (per RESEARCH Open Question 1 — dead-weight from PC import via the dead CsvImportModal path, out of scope for this cleanup)
- The store's raw-Firestore-doc read-fold (`data.teamTags` → `tags`) is deliberately left as-is since it operates on untyped snapshot data, not the `Song` type, and is the documented compatibility path for pre-existing Firestore docs that still physically carry a `teamTags` field (T-16.1-15, accepted risk)
- Where a test asserted team-tag-specific behavior that no longer has a corresponding code path (CsvImportModal's "computes song-level teamTags as union of arrangement teamTags"), the test was deleted rather than contorted to test dead functionality
- SongTable's team-pill-absence test was repointed to a positive assertion (a folded tag renders as an ordinary pill) since there's no longer a distinct team-pill code path to assert absence of
