# Phase 16.1 Plan 05: Song List Tags/Themes Split + Column Customization + VW Explainer Summary

**SongTable rebuilt with separate inline-editable Tags/Themes columns, a persistent column-visibility cog, VW-gated Category column, and a self-contained 1-2-3 methodology explainer — plus SongTable's first-ever component test.**

## Performance

- **Duration:** ~35 min
- **Completed:** 2026-07-13
- **Tasks:** 2 of 2 implementation tasks complete (Task 3 is a checkpoint:human-verify — see below)
- **Files modified:** 3 (1 created test, 1 created component, 1 modified component)

## Accomplishments

- Split the combined Tags/Themes/team-tag cell into two distinct columns: Themes (teal pills, inline add/remove) and Tags (pink user-tag pills, inline add/remove). Team pills are gone entirely — folded into `tags` upstream by plan 01's read-fold.
- Every non-Title data column (Category, Key, CCLI, Last Used, Tags, Themes) now gates on `songStore.columnVisibility.<key>`; Category additionally requires `authStore.vwModeEnabled` (D-16) so VW-off hides the whole column, not just the badge.
- Added a settings-cog popover (reusing the previously-empty trailing header cell) listing all six toggleable columns as checkboxes bound to `toggleColumn`, plus a `Reset` button bound to `resetColumns`. Title is intentionally absent from the list.
- Generalized the inline-tag editing trio (`openInlineEdit`/`commitInlineTag`/removal) over a `field: 'tags' | 'themes'` parameter so both columns share one implementation.
- Removing a Theme pill now also appends it to `removedThemes` via `updateSong`, completing the D-14 tie-in (upsertSongs already subtracts `removedThemes` from the incoming PC union on re-import).
- Added `VwExplainer.vue`: a self-contained popover teaching the app's existing 1-2-3 (Call to Worship / Intimate / Ascription), explicitly avoiding any 1-5 scale language (D-18). Triggered from a help icon next to the Category header, which itself only renders when VW mode is on.
- Added `src/components/__tests__/SongTable.test.ts` — the first component test for SongTable, covering column-visibility gating, VW-off Category hiding, and the Tags/Themes split with no team pills. Written RED-first (TDD), then made GREEN by the SongTable rework.

## Files Created/Modified

- `src/components/SongTable.vue` - Themes becomes its own inline-editable column (before Tags); Tags cell now only renders user tags; Category/Key/CCLI/LastUsed/Themes/Tags each gated by `columnVisibility`; Category additionally VW-gated; added column-visibility cog popover; generalized inline tag/theme editing; theme removal records `removedThemes`; added `themes` sort field/case.
- `src/components/VwExplainer.vue` - New self-contained 1-2-3 explainer popover (help icon + backdrop-dismissed panel), rendered next to the Category header.

## Decisions Made

- Reused the existing (previously empty) trailing header `<th>` as the cog trigger slot rather than inserting a new column — keeps table width unchanged and the cog visually anchored where users already look for row actions.
- VwExplainer manages its own `open` ref internally (no parent-controlled visibility prop) so it drops in with zero wiring beyond `<VwExplainer />` inside the Category header's flex row.
- Component test mocks the store modules directly via `vi.mock` rather than using `@pinia/testing`/`createTestingPinia`, since that package isn't a project dependency — this mirrors the precedent already established by `QuarterGrid.test.ts`.
