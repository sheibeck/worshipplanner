<objective>
Fold team tags into the flat `tags` set at the data layer and make PC import safe for the merged tags, plus stop user-removed themes from reappearing on re-import.

Purpose: This is the foundation wave for the widest-blast-radius decision (D-01). By read-normalizing `teamTags` into `tags` in the store's `subscribe()` and folding at the import/write layer, every downstream consumer that reads `song.tags` sees the merged set BEFORE any UI is repointed — so later waves can drop `teamTags` reads incrementally without data loss. The `Song.teamTags` field itself stays alive until plan 08 (type-removal cleanup) so intermediate waves keep `npm run type-check` green.

Output: `songs.ts` read-fold + import-fold, `pcSongImport.ts` mapping repoint, new `Song.removedThemes` tracking field, and store/util test coverage.
</objective>
