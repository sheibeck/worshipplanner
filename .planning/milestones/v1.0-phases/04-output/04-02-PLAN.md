CRITICAL from Research: Current Firestore rules require isOrgMember(orgId) to read service documents. Unauthenticated share viewers CANNOT satisfy this. Solution: embed a serviceSnapshot in the shareTokens/{token} document at creation time. ShareView reads ONLY the token document (public read: if true). Zero reads of the protected org collection.

BPM handling: At token creation time, resolve BPM for each song slot from songStore.songs by matching arrangement key to slot.songKey. Embed bpm into the snapshot so ShareView has complete data without any store subscriptions.

    ```typescript
    async function createShareToken(service: Service, orgIdValue: string): Promise<string> {
      // Generate cryptographically random 36-char hex token
      const array = new Uint8Array(18)
      crypto.getRandomValues(array)
      const token = Array.from(array, b => b.toString(16).padStart(2, '0')).join('')

      // Resolve BPM for each song slot from song store
      const songStore = useSongStore()
      const slotsWithBpm = service.slots.map(slot => {
        if (slot.kind === 'SONG' && slot.songId) {
          const song = songStore.songs.find(s => s.id === slot.songId)
          let bpm: number | null = null
          if (song) {
            const matchingArr = song.arrangements.find(a => a.key === slot.songKey)
            bpm = matchingArr?.bpm ?? song.arrangements[0]?.bpm ?? null
          }
          return { ...slot, bpm }
        }
        return slot
      })

      await setDoc(doc(db, 'shareTokens', token), {
        serviceId: service.id,
        orgId: orgIdValue,
        serviceSnapshot: {
          date: service.date,
          name: service.name,
          progression: service.progression,
          teams: service.teams,
          slots: slotsWithBpm,
          sermonPassage: service.sermonPassage,
          notes: service.notes,
          status: service.status,
        },
        createdAt: serverTimestamp(),
      })

      return token
    }
    ```

    Add `createShareToken` to the return object of the store.

    Note: `setDoc` is used (not `addDoc`) because we want the token string as the document ID, not an auto-generated ID. This is important because the URL path IS the token.
