  - `aiSuggestingAll = ref(false)` — loading state for Suggest All
  - `aiSongCache = ref(new Map<string, AiSongSuggestion[]>())` — session cache keyed by sermon context + slot VW type
  - `aiPerSlotLoading = ref(new Map<number, boolean>())` — per-slot loading for individual AI picks

**4. suggestAllSongs() Function:**
- Sets `aiSuggestingAll.value = true`.
- Iterates over `localService.slots`, finds all SONG slots that have no songId (empty).
- For each empty song slot, collects the `alreadySelectedSongs` from other non-empty song slots.
- For each empty song slot, calls `getSongSuggestions()` with: sermonTopic, sermonPassage, slot's requiredVwType, alreadySelectedSongs (accumulate — each call includes songs already accepted from prior slots in this batch), songs (from songStore.songs with only id/title/vwType/themes/lastUsedAt), recentServiceSongIds (collect from past 8 weeks of serviceStore.services).
- On success: take the first suggestion, resolve it against songStore to get title and key, and set it in `aiDraftSongs` map at that slot index.
- On null result: skip that slot (no draft).
- After all slots processed, sets `aiSuggestingAll.value = false`.

**5. fetchAiForSlot(slotIndex: number) Function:**
- Called by SongSlotPicker via emitted event (see below).
- Checks cache first. If cached, sets `aiPerSlotResults` for that slot index.
- If not cached, sets `aiPerSlotLoading` for that slot, calls `getSongSuggestions`, stores result in `aiPerSlotResults` and cache, clears loading.
- On error, sets `aiPerSlotError` for that slot.

**6. Accept/Reject AI Draft in Template:**
- In the song slot template (where slot.kind === 'SONG'), check if `aiDraftSongs.has(index)` for that slot.
- If an AI draft exists AND slot has no songId, show the draft inline:

  - Reason text in text-xs text-indigo-400
  - Green checkmark button (accept): calls `acceptAiSong(index)` which uses `onSelectSong(index, ...)` with the draft data, then deletes from aiDraftSongs
  - Gray X button (reject): calls `rejectAiSong(index)` which just deletes from aiDraftSongs
  - The draft display sits in the same area where the assigned song display would be (between the label row and the SongSlotPicker trigger).

**7. Pass AI data down to SongSlotPicker:**
- Add new optional props to the SongSlotPicker invocation:
  - `:aiSuggestions="aiPerSlotResults.get(index) ?? undefined"`
  - `:aiLoading="aiPerSlotLoading.get(index) ?? false"`
  - `:aiError="aiPerSlotError.get(index) ?? false"`
  - `:hasSermonContext="hasSermonContext"`
  - `@requestAiSuggestions="fetchAiForSlot(index)"`

**8. Recent service song IDs helper:**

**6. Resolve AI suggestions to Song objects (computed):**
```typescript
const resolvedAiSuggestions = computed(() => {
  if (!props.aiSuggestions) return []
  return props.aiSuggestions
    .map(ai => {
      const song = props.songs.find(s => s.id === ai.songId)
      return song ? { song, reason: ai.reason } : null
    })
    .filter((item): item is { song: Song; reason: string } => item !== null)
})
```

This keeps SongSlotPicker as a display component — it receives pre-fetched AI suggestions from the parent and just renders them. The parent (ServiceEditorView) handles the API calls and caching.
