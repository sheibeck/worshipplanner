4. Create `src/utils/claudeApi.ts` following the research patterns:
   - Lazy singleton `getClient()` with `dangerouslyAllowBrowser: true`
   - Use model `claude-haiku-3-5-20241022`
   - Export `AiSongSuggestion` interface: `{ songId: string, reason: string }`
   - Export `AiScriptureSuggestion` interface: `{ book: string, chapter: number, verseStart: number, verseEnd: number, reason: string, recentlyUsed: boolean, weeksAgoUsed: number | null }`
   - Export `safeParseJsonArray(text: string): unknown[] | null` — try JSON.parse first, then regex extract `\[[\s\S]*\]`, return null on all failures
   - Export `validateSongSuggestions(aiResult: AiSongSuggestion[], songs: { id: string }[]): AiSongSuggestion[]` — filter to only songIds that exist in the songs list
   - Export `validateScriptureSuggestions(aiResult: AiScriptureSuggestion[]): AiScriptureSuggestion[]` — filter to only books in BIBLE_BOOKS from `utils/scripture.ts`
   - Export `getSongSuggestions(params)` — builds prompt with sermon topic, passage, slot VW type, already selected songs, song library (id/title/vwType/themes/lastUsedAt only), recent service song IDs. System prompt instructs JSON-only response with exactly 3 items. Returns `AiSongSuggestion[] | null`. Catches all errors and returns null.
   - Export `getScriptureSuggestions(params)` — builds prompt with sermon topic, passage, natural language query, recent scriptures. System prompt instructs JSON-only response with 3-5 items. Returns `AiScriptureSuggestion[] | null`. Catches all errors and returns null.
   - Both functions: parse response with `safeParseJsonArray`, validate with the appropriate validator, return validated results. On any failure (API error, parse error, validation returns empty), return null.
   - `max_tokens: 512` for both calls.

5. System prompts (embed as constants in claudeApi.ts):
   - SONG_SYSTEM_PROMPT: Explain VW methodology (Type 1 = Call to Worship, Type 2 = Intimate, Type 3 = Ascription). Instruct: respond ONLY with valid JSON array, no markdown/prose/code fences. Return exactly 3 items matching `[{"songId":"<id>","reason":"<5-10 word reason>"}]`. Rules: prefer thematic matches, deprioritize songs used in last 2 weeks, consider already-selected songs for cohesion, songId must be exact from provided list.
   - SCRIPTURE_SYSTEM_PROMPT: Instruct: respond ONLY with valid JSON array, no markdown/prose/code fences. Return 3-5 items matching `[{"book":"<name>","chapter":<n>,"verseStart":<n>,"verseEnd":<n>,"reason":"<5-10 words>","recentlyUsed":<bool>,"weeksAgoUsed":<n|null>}]`. Rules: book names must match Protestant canon exactly, prefer passages thematically relevant to query/sermon, note if a passage was recently used.
