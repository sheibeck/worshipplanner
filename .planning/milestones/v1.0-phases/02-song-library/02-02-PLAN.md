  <files>src/components/SongSlideOver.vue, src/components/ArrangementAccordion.vue</files>
  <action>
    **ArrangementAccordion.vue**
    Collapsible section for one arrangement. Props: `arrangement: Arrangement`, `availableTags: string[]`. Emits: `update(arrangement)`, `remove`. Uses a local `isOpen` ref to toggle visibility. When open, shows editable fields:
    - Name (text input)
    - Key (text input or select with common keys: C, C#, D, Eb, E, F, F#, G, Ab, A, Bb, B, plus minor variants)
    - BPM (number input)
    - Length (number input, seconds — display as mm:ss)
    - Chord Chart URL (text input)
    - Notes (textarea)
    - Team Tags (toggle buttons for available tags — highlighted when selected)
    Header shows arrangement name + key + BPM summary when collapsed. Includes a "Remove" button (text-red-400) in the header.
    Dark palette: bg-gray-800 accordion body, border-gray-700 dividers, inputs match Song filter inputs.

    **SongSlideOver.vue**
    Right-side panel using `<Teleport to="body">`. Uses Vue `<Transition>` for slide-in/out animation (translate-x-full to translate-x-0).

    Props: `open: boolean`, `song: Song | null` (null = create mode, populated = edit mode).
    Emits: `close`, `saved`, `deleted`.

    Structure:
    - **Backdrop**: fixed inset-0 z-40 bg-black/30, click to close (semi-transparent, song list stays visible behind per Linear pattern)
    - **Panel**: fixed inset-y-0 right-0 z-50 w-[480px] (responsive: w-full on mobile, max-w-[480px]) bg-gray-900 border-l border-gray-800 shadow-2xl
    - **Header**: Title ("New Song" or "Edit Song") + Save button (indigo-600) + Cancel button (gray-700). Close X button.
    - **Scrollable body**: Form fields for song properties:
      - Title (required, text input)
      - CCLI Number (text input)
      - Author (text input)
      - VW Type selector: three buttons (1/2/3) using SongBadge-style colors, click to select, click again to deselect (set to null). Prominent visual selection state.
      - Themes (text input, comma-separated — split to array on save)
      - Notes (textarea)
      - Team Tags section: toggle buttons for predefined tags (Choir, Orchestra, Band) plus any custom tags. These are the song-level union tags.
    - **Arrangements section**: heading "Arrangements" with "Add Arrangement" button. List of ArrangementAccordion components. New arrangement gets a UUID via `crypto.randomUUID()`.
    - **Delete button** (edit mode only): at bottom, text-red-400 with "Delete Song" label. Clicking shows a confirmation inline or modal: "Are you sure you want to delete '{title}'? This cannot be undone." with Cancel and Delete buttons.

    On Save:
    - Validate title is not empty (show inline error if empty)
    - Compute song-level `teamTags` as union of all arrangement teamTags
    - In create mode: call `songStore.addSong(data)`, emit `saved`
    - In edit mode: call `songStore.updateSong(id, data)`, emit `saved`
    - On delete: call `songStore.deleteSong(id)`, emit `deleted`
    - Close panel after successful operation

    Use a local reactive copy of the song data for editing (deep clone on open). Save writes the local copy to store. Cancel discards local copy and closes. NO auto-save.

    All inputs follow dark palette: bg-gray-800 border-gray-700 text-gray-100 focus:ring-indigo-500 focus:border-indigo-500 placeholder-gray-500.
